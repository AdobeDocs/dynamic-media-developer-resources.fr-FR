# définir{#set}

Informations sur la visionneuse de médias.

req=set[,xml[, *`encoding`*]|{json[&amp;id=*`reqId`*]}]

Renvoie des informations sur les images, les vidéos, les échantillons et les différentes métadonnées associées à catalog::ImageSet pour l’entrée du catalogue d’images spécifiée dans le chemin d’accès de l’URL. Cette réponse est une structure hiérarchique déterminée par le type d’ensemble fourni. Une mise en forme appropriée est appliquée lorsque le format « xml » ou « json » est demandé.

La réponse HTTP peut être mise en cache avec la TTL basée sur `catalog::NonImgExpiration`.

>[!NOTE]
>
>Le caractère deux-points n’est pas autorisé dans les requêtes req=set.

Les requêtes qui prennent en charge le format de réponse JSON vous permettent de spécifier le nom du gestionnaire de rappel JS à l’aide de la syntaxe étendue `req=` paramètre :

`req=...,json [&handler = reqHandler ]`

`<reqHandler>` est le nom du gestionnaire JS présent dans la réponse JSONP. Seuls les caractères a-z, A-Z et 0-9 sont autorisés. Facultatif. La valeur par défaut est `s7jsonResponse`.
