# Notes de mise à jour des visionneuses Scene7 version 4.9.2{#scene-viewers-release-notes}

Améliorations, correctifs et problèmes connus dans les visionneuses Scene7 4.9.2

## Améliorations apportées aux visionneuses Scene7 4.9.2 {#section-0b0363fae0274737bace119f281e740f}

Les mises à niveau du lecteur sont rétrocompatibles et, par conséquent, aucune modification n’est nécessaire pour votre code Web existant. Toutefois, il est recommandé de tester les nouvelles visionneuses sur notre  d’évaluation. Contactez le support technique pour obtenir des instructions sur la manière d’accéder au serveur d’évaluation afin de pouvoir tester vos visionneuses. Une fois cette opération terminée, vous pouvez consulter votre site Web pour tester les mises à niveau.

* Configuration minimale requise améliorée pour les lecteurs vers iOS6.
* Ajout de la prise en charge du suivi des  personnalisés pour les lecteurs.
* Ajout de la prise en charge de la définition du débit initial pour la visionneuse de vidéos.
* La visionneuse de vidéos utilise désormais par défaut la diffusion en flux continu HLS sur le bureau Safari.
* Des info-bulles ont été reconfigurées pour résoudre divers bogues.
* Suppression des restrictions des fonctions sociales sur les visionneuses de vidéos et de catalogues électroniques des périphériques mobiles.

## Correctifs dans les visionneuses Scene7 4.9.2 {#section-405e47e202e54943bd843d82415c91cb}

Les bogues suivants sont maintenant corrigés :

* les visionneuses s’affichaient à une taille incorrecte après un retour du mode plein écran et un changement de fichier.
* les visiteurs affichaient le bouton de fermeture en mode Plein écran.
* La visionneuse de catalogue électronique n’affichait pas les zones cliquables en mode portrait sur les périphériques mobiles.
* La visionneuse de catalogue électronique affichait des boutons de panoramique sur les téléphones mobiles.
* Les info-bulles par défaut de la visionneuse de catalogue électronique ne s’appliquaient pas si l’ID  du n’était pas &quot;ecatalog&quot;.
* Les info-bulles de la visionneuse de catalogue électronique étaient masquées derrière les miniatures dans les  de grille.
* La visionneuse de catalogue électronique affichait un séparateur de page pour une seule page.
* Les zones cliquables de la visionneuse de catalogue électronique ne fonctionnaient pas sur IE9.
* La visionneuse de supports variés réinitialisait la position de la barre de défilement vidéo après le redimensionnement de la visionneuse.
* La visionneuse de supports variés utilisait une illustration incorrecte pour les boutons de panoramique à 360°.
* La vidéo de la visionneuse de supports variés présentait des info-bulles sous des échantillons de supports variés.
* Les boutons de rotation de la visionneuse de supports variés HTMl5 s’affichaient sur les tablettes.
* Les info-bulles de la visionneuse à 360° et de zoom ont été rognées par bord dans la visionneuse intégrée.
* La position de l’info-bulle Partage sur les réseaux sociaux a été déplacée pour les boutons de réseau social.
* Le format des info-bulles de partage sur les réseaux sociaux ne correspondait pas aux info-bulles de la visionneuse.
* Les info-bulles ne s’affichaient pas en mode plein écran sous Mac OS avec Safari 5.
* La taille de la visionneuse vidéo était incorrecte pour la barre de progression lors de la rotation de l’iPad et du basculement entre les modes d’écran.
* Par défaut, la visionneuse de vidéos générait des journaux de console.

## Problèmes connus dans les visionneuses Scene7 4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**Toutes les visionneuses Scene7**

* Les filigranes, l’obscurcissement et le verrouillage ne sont pas pris en charge.

**Toutes les visionneuses.**

* Les paramètres avec des noms d&#39;instance explicites dans le code doivent être remplacés ; les noms d’instance d’une URL doivent également être remplacés. Par exemple, `zoomView.iconeffect=0`.
* Le recadrage de la commande de diffusion d’images n’est pas pris en charge.
* Le bouton Fermer fonctionne uniquement si la visionneuse est ouverte dans une fenêtre enfant.
* Pour personnaliser le format d’info-bulle que vous ajoutez `!IMPORTANT` à la déclaration CSS.

**Visionneuse de catalogue électronique**

* Les modèles JavaScript dans les zones cliquables ne sont pas pris en charge.
* Si vous naviguez jusqu’à une autre page HTML, puis revenez parfois, le lecteur se réinitialise à la première page.
* Définissez le modificateur `ImageMapEffect` de survol sur `1` pour appeler les informations.

* `Frametransition` définie sur `none` ou `fade` n’est pas prise en charge.

* La mise en page s’affiche parfois incorrectement après rotation du périphérique iOS. Le zoom dans la page corrige la disposition.

**Visionneuse de supports variés**

* La lecture de la piste audio n’est pas prise en charge pour le moment.

**Lecteur Social**

* Pour rendre correctement les vignettes dans le courrier électronique sortant, le `serverurl` modificateur doit avoir une URL absolue.

**Visionneuse de vidéos**

* L’image d’affiche peut rencontrer l’erreur &quot;taille maximale&quot;. Il se peut que l’utilisateur doive augmenter le paramètre de limite pour la publication Image Serving.
* Les légendes vidéo nécessitent un jeu de règles  si elles hébergent une page HTML diffusée à partir d’un serveur externe qui n’est pas un serveur Scene7. Contactez l’assistance technique d’Adobe pour obtenir de l’aide.

**Visionneuses Flash AS3-all**

* encodez le caractère # dans les noms de fichier.
* La prise en charge du serveur ne parvient pas à charger les animations SWF avec les vidéos intégrées.
* Le support du serveur ne parvient pas à charger les habillages de la visionneuse s’il est compilé pour Flash Player 6. La solution consiste à compiler pour Flash Player 7.
* Macintosh OS et Flash Player version 10,0,32,18 : La solution dépend d’un pont JavaScript au lieu d’une connexion locale pour communiquer entre Flex et Flash. L’application Flex doit donc être intégrée dans l’enveloppe HTML.
* Actuellement, les visionneuses Flash prennent en charge les fichiers SWF uniquement pour Flash 7.
* En raison d’un problème avec Java v.1.5.0_06, le composant serveur n’est pas pris en charge avec cette version de Java. [http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990](http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990).
* Pour un paramètre d’URL personnalisée d’habillage, le paramètre de l’URL d’habillage doit être codé en URL.
* Les visionneuses d’images dont le nom contient des caractères ASCII étendus doivent être codées deux fois dans l’URL lorsqu’elles sont envoyées à la visionneuse ou la requête FVCTX échoue, car le lecteur Flash décode le nom avant de le transmettre au code de la visionneuse.
* Si la page HTML d’incorporation comporte un contenu important après la visionneuse, il est possible que le lecteur appelle une fonction JavaScript avant que la page ne soit complètement chargée. Cela peut annuler le chargement des pages. Il est conseillé d’utiliser un gestionnaire de  onLoad pour retarder l’initialisation du lecteur jusqu’à la fin du chargement de la page.
* Les modifications apportées au `S7Config.setFlashParam()` permettent de transmettre plusieurs paramètres/valeurs avec ceci `syntax S7Config.setFlashParam(<viewer ID>, <parameter name>, <parameter value> [, <parameter name>, <parameter value>])`, mais dans ce cas, le premier paramètre à transmettre doit être le paramètre &quot;image&quot; suivi du paramètre &quot;currentFrame&quot; (si nécessaire), puis tout autre paramètre.

* Lorsque vous utilisez le `skipFrames=frameList` paramètre pour omettre des images, le lecteur renvoie une erreur si une image omise est appelée directement par la fonction d’image de modification JavaScript `InitialFrame`ou dans le cas d’une visionneuse de rendus à laquelle sont associées deux nuances ou plus d’une image, lorsque la combinaison d’échantillons référençant l’image omise est sélectionnée.

* Le recadrage n’est pas pris en charge conjointement avec les  de zoom ou les échantillons dans les `image=` arguments ou `modifier=` .

* L’ `rgn` argument n’est pas pris en charge pour les visionneuses Flash pour le modificateur d’image.
* Un nouveau correctif de sécurité IE demande à l’utilisateur d’activer Flash et d’autres applications ActiveX dans IE. Pour plus d&#39;informations sur les solutions, consultez le site de Microsoft.
* Les problèmes de sécurité empêchent le lecteur de lire depuis un serveur différent sur le même domaine, sauf si les deux URL sont des noms de domaine complets ou si la &quot;stratégie interdomaines&quot; est spécifiquement définie pour autoriser l’accès à partir de ce serveur.
* Avec les nouvelles fonctionnalités de sécurité de Flash 7, si les visionneuses tentent de lire à partir d’un serveur différent sur le même domaine depuis l’emplacement des visionneuses (c’est-à-dire qu’elles sont redirigées par les chemins dans le `infoServerUrl`, `searchEngineUrl`ou le `serverUrl parameters`), une alerte de sécurité avertit l’utilisateur de ce fait et vous demande si vous voulez autoriser cela, à moins que la &quot;stratégie inter-domaines&quot; ne soit définie pour autoriser l’accès à partir de ce serveur.

   Pour plus d’informations sur la configuration d’une &quot;stratégie inter-domaines&quot;, voir l’article suivant : [http://kb2.adobe.com/cps/142/tn_14213.html](http://kb2.adobe.com/cps/142/tn_14213.html).

* Erreur d’exception générée lorsque le lecteur est incorporé à la page HTTPS. Le problème est dû à la communication avec la journalisation du serveur. Pour plus d’informations : [http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html](http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html)
