# Notes De Mise À Jour Des Visionneuses Scene7 4.9.2{#scene-viewers-release-notes}

Améliorations, correctifs de bugs et problèmes connus dans les visionneuses Scene7 4.9.2

## Améliorations apportées aux visionneuses Scene7 4.9.2 {#section-0b0363fae0274737bace119f281e740f}

Les mises à niveau des visionneuses sont rétrocompatibles. Aucune modification n’est donc nécessaire au code web existant. Cependant, il est recommandé de tester les nouvelles visionneuses dans l’environnement d’évaluation d’Adobe. Contactez l’assistance technique pour obtenir des instructions sur la manière d’accéder au serveur d’évaluation afin de pouvoir tester vos visionneuses. Une fois l’activation terminée, vous pouvez consulter votre site web pour tester les mises à niveau.

* Augmentation des exigences minimales pour les visiteurs d’iOS6.
* Ajout de la prise en charge du suivi des événements personnalisés pour les visionneuses.
* Ajout de la prise en charge de la définition du débit initial de la visionneuse de vidéos.
* La visionneuse de vidéos utilise désormais par défaut la diffusion en continu d’HLS sur le bureau Safari.
* Infobulles refactorisées pour résoudre divers bogues.
* Suppression des restrictions des fonctionnalités sociales sur les visionneuses de vidéos et de catalogues électroniques des appareils mobiles.

## Correctifs de bugs dans les visionneuses Scene7 4.9.2 {#section-405e47e202e54943bd843d82415c91cb}

Les bogues suivants ont été corrigés :

* la taille d’affichage des visionneuses était incorrecte après le retour du mode plein écran et la permutation des ressources.
* Les visionneuses affichaient le bouton Fermer en mode Plein écran.
* La visionneuse de catalogue électronique n’affichait pas les zones cliquables en mode portrait sur les appareils mobiles.
* La visionneuse de catalogue électronique affichait des boutons de panoramique sur les téléphones mobiles.
* Les info-bulles par défaut de la visionneuse de catalogue électronique ne s’appliquaient pas si l’ID de conteneur n’est pas « catalogue électronique ».
* Les info-bulles de la visionneuse de catalogue électronique étaient masquées derrière des miniatures en mode Grille.
* La visionneuse de catalogue électronique affichait un séparateur de pages pour une seule page.
* Échec du fonctionnement des zones cliquables de la visionneuse de catalogue électronique sur IE9.
* La visionneuse de médias mixtes réinitialisait la position de la barre vidéo après avoir redimensionné la visionneuse.
* La visionneuse de médias mixtes utilisait une illustration incorrecte pour les boutons de rotation.
* Vidéo de visionneuse de médias mixtes affichant des info-bulles sous des échantillons de médias mixtes.
* Les boutons de rotation de la visionneuse de médias mixtes HTMl5 s’affichaient sur les tablettes.
* Les infobulles des visionneuses Rotation et Zoom étaient tronquées par leur contour dans la visionneuse incorporée.
* La position de l’info-bulle Partage social a été déplacée pour les boutons sociaux.
* Le format des infobulles de partage sur les réseaux sociaux ne correspondait pas aux infobulles de la visionneuse.
* Les info-bulles ne s’affichaient pas en mode plein écran sur macOS avec Safari 5.
* La taille de la visionneuse vidéo était incorrecte pour la barre de progression lors de la rotation d’iPad et du basculement entre les modes d’écran.
* La visionneuse de vidéos générait les journaux de la console par défaut.

## Problèmes connus dans les visionneuses Scene7 4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**Toutes les visionneuses Scene7**

* Les filigranes, l’obfuscation et le verrouillage ne sont pas pris en charge.

**Tous les observateurs**

* Les paramètres dont le code contient des noms d’instance explicites doivent être remplacés. Les noms d’instance d’une URL doivent également être remplacés. Par exemple, `zoomView.iconeffect=0`.
* Le recadrage de commande de diffusion d’images n’est pas pris en charge.
* Le bouton Fermer ne fonctionne que si la visionneuse est ouverte dans une fenêtre enfant.
* Pour personnaliser le format des infobulles, ajoutez des `!IMPORTANT` à la déclaration CSS.

**visionneuse de catalogue électronique**

* Les modèles JavaScript dans les zones cliquables ne sont pas pris en charge.
* Lorsque vous accédez à une autre page HTML, puis revenez de temps en temps, la visionneuse se réinitialise à la première page.
* Définissez `ImageMapEffect` modificateur de survol sur `1` pour appeler infopanels.

* `Frametransition` valeur définie sur `none` ou `fade` n’est pas prise en charge.

* La mise en page s’affiche parfois incorrectement après la rotation de l’appareil iOS. Le zoom avant sur la page corrige la disposition.

**Visionneuse de médias mixtes**

* La lecture de la piste audio n’est actuellement pas prise en charge.

**Observateur de réseaux sociaux**

* Pour effectuer correctement le rendu des miniatures dans l’e-mail sortant, le modificateur `serverurl` doit comporter une URL absolue.

**Visionneuse vidéo**

* L’image d’affiche peut rencontrer une erreur de « taille maximale ». L’utilisateur ou l’utilisatrice doit augmenter le paramètre de limite pour la Publication du service d’image.
* Les sous-titres vidéo nécessitent un ensemble de règles d’entreprise s’ils hébergent une page HTML diffusée à partir d’un serveur externe qui n’est pas un serveur Scene7. Contactez l’assistance technique d’Adobe pour obtenir de l’aide.

**Visionneuses Flash AS3-all**

* Codez deux fois le caractère # dans les noms de ressources.
* La prise en charge du serveur ne parvient pas à charger les animations SWF avec les vidéos incorporées.
* La prise en charge du serveur ne parvient pas à charger les habillages de la visionneuse s’ils sont compilés pour Flash Player 6. Vous pouvez compiler pour Flash Player 7.
* Macintosh OS et Flash Player version 10,0,32,18 : la solution de contournement repose sur un pont JavaScript au lieu d&#39;une connexion locale pour communiquer entre Flex et Flash. L’application Flex doit donc être intégrée dans le wrapper HTML.
* Actuellement, les visionneuses Flash prennent en charge les SWF conformes pour Flash 7 uniquement.
* En raison d’un problème avec Java™ v.1.5.0_06, le composant serveur n’est pas pris en charge avec cette version de Java™. [&#128279;](https://bugs.java.com/?bug_id=6274990).
* Pour un paramètre d’URL d’habillage personnalisé, le paramètre de l’URL d’habillage doit être encodé en URL.
* Si la page HTML d’incorporation comporte du contenu significatif après la visionneuse, il est possible que cette dernière appelle une fonction JavaScript avant que la page ne soit complètement chargée. Cette action peut annuler le chargement de la page. Une solution de contournement consiste à utiliser un gestionnaire d’événements onLoad pour retarder l’initialisation de la visionneuse jusqu’à ce que le chargement de la page soit terminé.
* Lors de l’utilisation du paramètre `skipFrames=frameList` pour omettre des images, la visionneuse renvoie une erreur si une image omise est appelée directement par la fonction de modification d’image de JavaScript `InitialFrame` ou si un RenderSet qui comporte deux échantillons ou plus associés à une image, lorsque la combinaison d’échantillons référençant l’image omise est sélectionnée.

* Le recadrage n’est pas pris en charge avec les cibles de zoom ou les échantillons dans les arguments `image=` ou `modifier=`.

* L’argument `rgn` n’est pas pris en charge pour les visionneuses Flash pour le modificateur d’image.
* Un nouveau correctif de sécurité IE nécessite que l&#39;utilisateur active Flash et d&#39;autres applications ActiveX® dans IE. Consultez le site de ® pour plus d’informations sur les solutions de contournement.
* Des problèmes de sécurité empêchent la visionneuse de lire à partir d’un autre serveur du même domaine, sauf si les deux URL sont des noms de domaine complets ou si la « politique interdomaines » est définie pour autoriser l’accès à partir de ce serveur.


  Pour plus d’informations sur la configuration d’une « politique interdomaines », consultez l’article suivant : [&#128279;](https://kb2.adobe.com/cps/142/tn_14213.html).

* Exception générée lorsque la visionneuse est incorporée dans une page HTTPS. Le problème est dû à la communication avec la journalisation du serveur. Pour plus d&#39;informations [&#128279;](https://www.adobe.com/fr/products/flashplayer/end-of-life.html).
