# Notes de mise à jour de Scene7 Viewers 4.9.2{#scene-viewers-release-notes}

Améliorations, correctifs de bogues et problèmes connus dans Scene7 Viewers 4.9.2

## Améliorations apportées aux visionneuses Scene7 4.9.2 {#section-0b0363fae0274737bace119f281e740f}

Les mises à niveau du lecteur sont rétrocompatibles et, par conséquent, aucune modification n’est nécessaire à votre code Web existant. Cependant, il est recommandé de tester les nouvelles visionneuses sur notre environnement d’évaluation. Contactez l’assistance technique pour obtenir des instructions sur la façon d’accéder au serveur d’évaluation afin que vous puissiez tester vos visionneuses. Une fois cette opération effectuée, vous pouvez consulter votre site Web pour tester les mises à niveau.

* Configuration minimale requise améliorée pour les lecteurs iOS6.
* Prise en charge Ajoutée du suivi de événement personnalisé pour les visionneuses.
* Prise en charge Ajoutée de la définition du débit initial pour la visionneuse de vidéos.
* La visionneuse de vidéos utilise désormais par défaut la diffusion en flux continu HLS sur un ordinateur de bureau Safari.
* Des info-bulles ont été refaçonnées afin de résoudre divers bogues.
* Suppression des restrictions relatives aux fonctions sociales sur les visionneuses de vidéos et de catalogues électroniques des périphériques mobiles.

## Correctifs de bogues dans Scene7 Viewers 4.9.2 {#section-405e47e202e54943bd843d82415c91cb}

Les bogues suivants sont maintenant corrigés :

* les visionneuses s’affichaient à une taille incorrecte après un retour du mode plein écran et un changement de fichier.
* les lecteurs affichaient le bouton de fermeture en mode plein écran.
* La visionneuse de catalogue électronique n’affichait pas les zones cliquables en mode portrait sur les périphériques mobiles.
* La visionneuse de catalogue électronique affichait des boutons panoramiques sur les téléphones mobiles.
* Les info-bulles par défaut de la visionneuse de catalogue électronique ne s’appliquaient pas si l’ID de conteneur n’était pas &quot;catalogue électronique&quot;.
* Les info-bulles de la visionneuse de catalogue électronique étaient masquées derrière les miniatures dans la vue de grille.
* La visionneuse de catalogue électronique affichait un séparateur de page pour une seule page.
* Les zones cliquables de la visionneuse de catalogue électronique ne fonctionnaient pas sur IE9.
* La visionneuse de supports variés réinitialisait la position de défilement de la vidéo après avoir redimensionné la visionneuse.
* La visionneuse de supports variés utilisait une illustration incorrecte pour les boutons de panoramique à 360°.
* La vidéo de la visionneuse de supports variés présentait des info-bulles sous des échantillons de médias mixtes.
* Les boutons de rotation de la visionneuse de supports variés HTMl5 s’affichaient sur les tablettes.
* Les info-bulles de la visionneuse à 360° et de zoom ont été tronquées par bord dans la visionneuse intégrée.
* La position de l’info-bulle de partage sur les réseaux sociaux a été déplacée.
* Le format des info-bulles de partage sur les réseaux sociaux ne correspondait pas aux info-bulles de la visionneuse.
* Les info-bulles ne s’affichaient pas en mode plein écran sous Mac OS avec Safari 5.
* La taille de la visionneuse de vidéos était incorrecte pour la barre de progression lors de la rotation de l’iPad et du basculement entre les modes d’écran.
* Par défaut, la visionneuse de vidéos générait des journaux de console.

## Problèmes connus dans Scene7 Viewers 4.9.2 {#section-368dd0dc138b44e28b49314e4ad0fa1a}

**Toutes les visionneuses Scene7**

* Les filigranes, l’obscurcissement et le verrouillage ne sont pas pris en charge.

**Toutes les visionneuses.**

* Les paramètres avec des noms d&#39;instance explicites dans le code doivent être remplacés ; les noms d’instances d’une URL doivent également être remplacés. Par exemple, `zoomView.iconeffect=0`.
* Le recadrage de la commande Image Serving n’est pas pris en charge.
* Le bouton Fermer ne fonctionne que si la visionneuse est ouverte dans une fenêtre enfant.
* Pour personnaliser le format d’info-bulle, ajoutez `!IMPORTANT` à la déclaration CSS.

**Visionneuse de catalogue électronique**

* Les modèles JavaScript dans les zones cliquables ne sont pas pris en charge.
* Si vous naviguez jusqu’à une autre page HTML, puis revenez parfois, la visionneuse revient à la première page.
* Définissez le modificateur de survol `ImageMapEffect` sur `1` pour appeler des informations.

* `Frametransition` définie sur  `none` ou n’ `fade` est pas prise en charge.

* La mise en page s’affiche parfois incorrectement après rotation du périphérique iOS. Le zoom dans la page corrige la disposition.

**Visionneuse de supports variés**

* Pour le moment, la lecture de piste audio n’est pas prise en charge.

**Visionneuse de réseaux sociaux**

* Pour afficher correctement les miniatures dans le courrier électronique sortant, le modificateur `serverurl` doit avoir une URL absolue.

**Visionneuse de vidéos**

* L’image d’affiche peut rencontrer l’erreur &quot;taille maximale&quot;. L’utilisateur doit peut-être augmenter le paramètre de limite pour la publication Image Serving.
* Les légendes vidéo nécessitent un jeu de règles de société si elles hébergent une page HTML diffusée à partir d’un serveur externe qui n’est pas un serveur Scene7. Contactez l&#39;assistance technique de l&#39;Adobe pour obtenir de l&#39;aide.

**Flash des visionneuses AS3-all**

* Doublon codez le caractère # dans les noms de fichier.
* Le support serveur ne parvient pas à charger les animations SWF avec les vidéos intégrées.
* Le support serveur ne parvient pas à charger les habillages de la visionneuse s’ils sont compilés pour le Flash Player 6. La solution consiste à compiler pour le Flash Player 7.
* Macintosh OS et version de Flash Player 10,0,32,18 : La solution dépend d’un pont JavaScript au lieu d’une connexion locale pour communiquer entre Flex et le Flash. L’application Flex doit donc être intégrée dans l’enveloppe HTML.
* Actuellement, les visionneuses de Flash prennent en charge les fichiers SWF uniquement respectés pour le Flash 7.
* En raison d’un problème avec Java v.1.5.0_06, le composant serveur n’est pas pris en charge avec cette version de Java. [http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990](http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6274990).
* Pour un paramètre d’URL personnalisée d’habillage, le paramètre de l’URL d’habillage doit être codé en URL.
* Les visionneuses d’images dont le nom contient des caractères ASCII étendus doivent être codées deux fois en URL lorsqu’elles sont envoyées à la visionneuse ou la demande FVCTX échoue car le lecteur de Flash décode le nom avant de le transmettre au code de la visionneuse.
* Si la page HTML incorporée comporte un contenu important après la visionneuse, il est possible que celle-ci appelle une fonction JavaScript avant que la page ne soit complètement chargée. Cela peut arrêter le chargement des pages. Il est conseillé d’utiliser un gestionnaire de événement onLoad pour retarder l’initialisation de la visionneuse jusqu’à ce que le chargement de la page soit terminé.
* Les modifications apportées à `S7Config.setFlashParam()` permettent de transmettre plusieurs paramètres/valeurs avec cette `syntax S7Config.setFlashParam(<viewer ID>, <parameter name>, <parameter value> [, <parameter name>, <parameter value>])`, mais dans ce cas, le premier paramètre à transmettre doit être le paramètre &quot;image&quot; suivi du paramètre &quot;currentFrame&quot; (si nécessaire), puis tout autre paramètre.

* Lors de l’utilisation du paramètre `skipFrames=frameList` pour omettre des images, le lecteur renvoie une erreur si une image omise est appelée directement par la fonction d’image de modification JavaScript `InitialFrame`, ou dans le cas d’une visionneuse de rendu à laquelle sont associées deux nuances ou plus d’une image, lorsque la combinaison d’échantillons référençant l’image omise est sélectionnée.

* Le recadrage n’est pas pris en charge conjointement avec les cibles de zoom ou les nuances dans les arguments `image=` ou `modifier=`.

* L&#39;argument `rgn` n&#39;est pas pris en charge pour les visionneuses Flash pour le modificateur d&#39;image.
* Un nouveau correctif de sécurité IE demande à l&#39;utilisateur d&#39;activer le Flash et d&#39;autres applications ActiveX dans IE. Consultez le site de Microsoft pour plus d&#39;informations sur les solutions.
* Les problèmes de sécurité empêchent le lecteur de lire depuis un serveur différent sur le même domaine, sauf si les deux URL sont des noms de domaine complets ou si la &quot;stratégie interdomaines&quot; est spécifiquement définie pour autoriser l’accès depuis ce serveur.
* Avec les nouvelles fonctionnalités de sécurité du Flash 7, si les lecteurs tentent de lire à partir d&#39;un serveur différent sur le même domaine d&#39;où se trouvent les lecteurs (c&#39;est-à-dire qu&#39;ils sont redirigés par les chemins d&#39;accès dans `infoServerUrl`, `searchEngineUrl` ou `serverUrl parameters`), une alerte de sécurité avertit l&#39;utilisateur de ce fait et vous demande si vous souhaitez autoriser cet accès, sauf si la &quot;stratégie interdomaines&quot; est définie pour autoriser l&#39;accès à partir de ce serveur.

   Pour plus d’informations sur la configuration d’une &quot;stratégie interdomaines&quot;, voir l’article suivant : [http://kb2.adobe.com/cps/142/tn_14213.html](http://kb2.adobe.com/cps/142/tn_14213.html).

* Erreur d’exception générée lorsque le lecteur est incorporé à la page HTTPS. Le problème est dû à la communication avec la journalisation du serveur. Pour plus d’informations : [http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html](http://helpx.adobe.com/flash-player/kb/flash-player-issues-secure-sockets.html)
