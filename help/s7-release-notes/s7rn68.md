## Faits saillants des versions {#section-524cffc79c1f4bb38627a5ecccd0bf52}

* Ajout de la prise en charge du responsive design web : diffusion d’images réactive basée sur des points d’arrêt et des visionneuses HTML5 réactives (visionneuses d’images, vidéo, supports variés, catalogue électronique, visionneuses à 360°, visionneuses de nuanciers, zoom) qui se redimensionnent dynamiquement en fonction de la mise en page.
* Intégration du rapport vidéo Adobe Analytics.
* Personnalisation avancée de la visionneuse HTML5 : créez des interactions de visionneuse personnalisées avec la page web d’incorporation.
* Amélioration des images : possibilité de recadrer des images sur un chemin incorporé lors de la diffusion.

## Visionneuses {#section-609885a9c8514a4095da74a0b526138d}

Les éléments suivants ont été mis à jour :

* La configuration minimale requise pour le système d’exploitation Windows est désormais Windows 7.
* La configuration minimale requise pour Internet Explorer est désormais la version 9.
* La configuration système minimale requise pour macOS est désormais macOS 10.8.
* La configuration minimale requise pour Safari est désormais la version 6.

Les éléments suivants ont été refactorisés :

* Visionneuses HTML5 pour la prise en charge du responsive design.
* Les visionneuses HTML5 peuvent utiliser des marqueurs de taille et d’entrée CSS pour contrôler l’affichage sur les ordinateurs de bureau et les appareils mobiles.

Les éléments suivants sont désormais pris en charge :

* Intégration des rapports vidéo Adobe Analytics pour la visionneuse de vidéos HTML5 et la visionneuse de médias mixtes.
* Suivi des événements de plusieurs visionneuses HTML5 sur la même page.
* Ajout de `SetIndicator` à la visionneuse de zoom HTML5 et à la visionneuse de médias mixtes HTML5 pour représenter visuellement des échantillons sur les appareils mobiles.
* Ajout d’une API `getComponent` pour autoriser l’accès externe aux composants de la visionneuse HTML5.
* Paramètres de configuration de la visionneuse HTML5 transmis en tant qu’objet JSON unique.
* Légendes WebVTT à transmettre en tant qu’objet JSON, ce qui élimine la nécessité de configurer un ensemble de règles.
* Modèles JavaScript dans la visionneuse de catalogue électronique HTML5 pour lier les zones cliquables à des pages externes.
* Liaison à des pages dans la visionneuse de catalogue électronique HTML5.
* `stagesize` à la visionneuse déroulante HTML5.
* `initComplete` gestionnaire aux visionneuses HTML5 pour le déclenchement de fonctions après les visionneuses.
* Gestionnaire Analytics vers les visionneuses HTML5 pour la prise en charge du suivi tiers.
* Ajout du bouton « Tout sélectionner » pour faciliter l’obtention d’un lien et d’un code intégré pour le partage sur les réseaux sociaux.
* Ajout d’un redimensionnement pour FlyoutZoomView.
* Définition des points d’arrêt de chargement d’image pour FlyoutZoomView en cas d’utilisation avec le responsive design.
* Ajout de la prise en charge des clics pour le réglage du volume.
* SpinView et ZoomView pour appliquer des mouvements à l’incorporation d’une page web lors de l’interaction avec une visionneuse.
* Ajout de l’API `unload()` à SpinView et ZoomView pour permettre l’élimination du composant au moment de l’exécution.
* La visionneuse de vidéos permet d’appliquer des mouvements à la page web d’incorporation lorsque vous n’interagissez pas avec la visionneuse.
* Définition de la zone de zoom initiale pour ZoomView.
* ZoomView pour masquer `IconEffect` composant lorsque le zoom est impossible

Voir aussi [Guide de référence des visionneuses](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/homeviewers.html?lang=fr).

## Imagerie {#section-113f27ea04cd4998a324bba71519c93c}

* Prise en charge de Progressive JPEG.
* Modification des paramètres par défaut de masquage flou pour les nouveaux clients.

## Plateforme {#section-7e2e7bd11b81424e8cd939b6675ca729}

* Les fichiers entrants sont désormais vérifiés pour les virus à l’aide de Clam AntiVirus avant l’ingestion. Tous les fichiers ayant échoué sont immédiatement mis en quarantaine.
* Étendez le pop-up des détails de la tâche pour inclure l’heure de chaque enregistrement de détail.
* API implémentée pour supprimer une valeur de métadonnées d’une ressource (deleteAssetMetadata).
* Modification du comportement de recherche par défaut pour inclure dc.subject.
* Modification du mécanisme de réinitialisation du mot de passe - Mot de passe oublié - en un processus à plusieurs étapes pour éviter le DOS.
* Pour éviter toute liste bloquée, convertissez l’e-mail de mot de passe oublié d’HTML en texte brut.
* Suppression du rôle par défaut.
* Outre la possibilité de charger du contenu vers Scene7 Publishing System par le biais du chargement de répertoire serveur (ftp) et de charger des URL, vous pouvez désormais également charger du contenu à l’aide de HTTP POST. Pour plus d’informations sur l’utilisation du POST HTTP[ reportez-vous à la section ](/help/aem-ips-api/c-overview.md)API Image Production System.

## Correctifs de bugs {#section-456515477c8a4aa69309de43292ce5ed}

**Visionneuses**

* Problèmes de sécurité potentiels liés au zoom rapide Flash et aux visionneuses Fenêtre déroulante.
* Les visionneuses HTML5 ne se chargent pas en raison d’un UserAgent non reconnu.
* Glisser-déposer la souris de la visionneuse de catalogue électronique HTML5 ne fonctionne pas dans Internet Explorer avec la page contenant des zones cliquables.
* La visionneuse de vidéos HTML5 empêche l’accès pour débloquer le plug-in Flash.
* Les visionneuses HTML5 empêchent l’impression sur le navigateur Firefox.
* La visionneuse Zoom d’HTML5 affiche une légère gigue lors du lancement ou de la réinitialisation du zoom.
* Prise en charge HTTPS des visionneuses HTML5 et AS3.
* autoResize ne doit pas se trouver dans l’URL de copie/code intégré de la visionneuse Fenêtre déroulante HTML.
* Aucun modificateur videoserverurl dans l’URL de copie de la visionneuse de vidéos HTML5.
* Le paramètre `contenturl` provient de l’URL d’aperçu de la visionneuse HTML5 du serveur de publication.

**Imagerie**

* Impossible d’ajouter la vue ajustée en tant qu’image par défaut dans la configuration de la publication d’images.
* Remplacez le libellé colorManagementOptions « Convertir en SRGB (par défaut) » par « Gestion des couleurs par défaut ».
* Lorsque vous sélectionnez plusieurs ressources d’image dans Fichier > Modifier les informations , les informations de modification n’étaient pas disponibles.

**Platform**

* Ajout d’un message d’alerte lorsque vous modifiez la configuration d’Adobe Analytics, puis tentez d’échapper la configuration sans enregistrer.
* La référence des options du projet est réinitialisée sur Aucune lors de la modification d’une tâche de chargement planifiée.
* Le dossier File-Move ne doit pas être affiché pour le rôle Utilisateur.
* Impossible de voir certains onglets dans la vue Détails.
* Erreur de modification web-to-print lorsque la version de débogage de Flashplayer est installée.
* « Espace après » ne fonctionne pas ; il doit utiliser `sa`, actuellement il est `margb`.
* Lors de l’initialisation des paramètres de l’entreprise, Scene7 Publishing System envoie des `<defaultValue>` d’option en double dans l’API `<createMetadataField>`.

* Le plug-in Flash et le système de publication Scene7 se sont bloqués en raison de nombreux dossiers ne disposant pas de suffisamment de mémoire Flash.
* Lors de la suppression de la valeur du champ Date/Heure défini par l’utilisateur, le système de publication Scene7 n’envoie pas de requête à IPS (Image Publishing System).
* Si les options de codage vidéo adaptative sont activées, le message d’avertissement Codage vidéo adaptative s’affiche lors du retraitement d’un fichier PDF ou PSD.
* Échec de l’exportation de la ressource lorsque SSL est activé.
* Un mot de passe incorrect était affiché dans l’e-mail Mot de passe oublié.
* Toutes les demandes d’administration web du système de publication Scene7 sont redirigées vers HTTP.
* Tous les paramètres prédéfinis de lot étaient appliqués lors du chargement si aucun paramètre prédéfini n’était sélectionné.
* Le système de publication Scene7 demande s7info par le biais du protocole HTTP lors du chargement du contenu s7info.
* L’URL mentionnée dans l’e-mail Mot de passe oublié est HTTP, même si elle provient du système de publication SSL Scene7.
* Les caractères spéciaux étaient codés dans un e-mail « Mot de passe oublié » au format HTML, ce qui entraînait l’affichage incorrect du mot de passe.
* Les informations du compte FTP Scene7 sont manquantes dans l’e-mail de bienvenue pour un compte d’évaluation
* Échec de la tâche de retraitement lorsque les options de projet et de recadrage sont sélectionnées.
* Le fichier de chargement ne fonctionne pas lorsque les options de tâche sont réinitialisées aux valeurs par défaut de l’entreprise.
* Le libellé Société ne s’affiche pas complètement dans la page Configuration d’Adobe Analytics.
* Impossible d’importer les métadonnées.

## Problèmes connus {#section-b2608177928b4174b080d89c938bb0d3}

* La mise à jour de mai 2014 d’Adobe Illustrator et d’Adobe InDesign devrait introduire des incompatibilités avec les plug-ins à l’origine de Scene7 FXG. Ces incompatibilités affectent les utilisateurs de la solution Scene7 Web-to-Print. Si vous utilisez Scene7 Web-to-Print, ne mettez pas à niveau Illustrator et InDesign vers cette mise à jour de Creative Cloud. Continuez plutôt à utiliser la version antérieure à mai 2014. Adobe Inc prévoit la restauration de la compatibilité de la création de Scene7 FXG avec des versions plus récentes d’Illustrator et d’InDesign pour l’automne 2014.
* Le bouton Code incorporé ne fonctionne pas lorsque le paramètre prédéfini de la visionneuse universelle contient un paramètre prédéfini AS2 désactivé par défaut.
* Le retraitement de la visionneuse de vidéos adaptative crée toujours une visionneuse de vidéos adaptative.
* L’interface utilisateur de la boîte de dialogue Détails de la tâche est tronquée à droite après l’ajout de l’option Heure dans les détails.

## Fonctionnalités non prises en charge {#section-7b04b6fdb8d1473880ccaf84211254ad}

* La césure du texte n’est plus prise en charge dans la version d’automne de Scene7 6.9. Les dictionnaires de césure définissent la manière dont les mots peuvent être fractionnés et cédés pour prendre en charge un flux de texte optimal.
* À compter du 31 janvier 2014, Scene7 a officiellement arrêté la prise en charge de la plateforme de la visionneuse DHTML.
* Les bannières dynamiques (Flash) sont abandonnées dans cette version. L’adoption de cette fonctionnalité a été faible, car d’autres technologies ont émergé pour créer, gérer et diffuser des bannières de site web personnalisées. La fonctionnalité de bannière dynamique (Flash) Scene7 continue de fonctionner côté serveur, mais elle n’est plus prise en charge.
* Rappel : le serveur de métadonnées et l’utilisation des API associées ont été officiellement supprimés de la prise en charge dans la version précédente de Scene7.
* Les extensions de la suite créative Scene7 (panneaux Flash) ont été abandonnées fin 2014. D’autres développements des workflows d’ingestion de contenu se concentrent sur Adobe Experience Manager Dynamic Media et Creative Cloud. Pour Scene7, les fonctionnalités de chargement et d’exportation du système de publication Scene7 restent et peuvent être utilisées à la place.
* Depuis mai 2014, Akamai ne prend plus en charge les notifications par e-mail lorsque l’invalidation du cache dans Scene7 est terminée. Le temps de traitement standard de l’invalidation du réseau CDN est d’environ huit minutes. S’il existe une grande file d’attente de requêtes ou de nombreuses URL, le temps de traitement peut être légèrement plus long.
* Catégorie « Paramètres prédéfinis universels » dans l’onglet URL et Code intégré dans la vue Détails.
* Diffusion en continu RTMP pour la vue détaillée de la vidéo.
* « Communications par e-mail » et servlet SendMailSimple.jsp du système de publication Scene7.
* Section du compte FTP Scene7 dans le modèle de courrier électronique de bienvenue pour l’utilisateur du portail multimédia.
* dc.title à partir du panneau de filtrage, car il provoquait la panne de l’instance IPS (Image Publishing System).
* Les fonctionnalités de réseaux sociaux s’affichent en mode plein écran sur les visionneuses HTML5.
* Le bouton Fermer s’affiche en mode plein écran sur les visionneuses HTML5.

## Nouvelles fonctionnalités et correctifs de bugs dans Scene7 SDK 2.7.1 {#section-b82b4f546ce44a769e94fb1493511035}

**Général**

* Augmentation des exigences minimales pour iOS6.
* Correction : les infobulles de Safari étaient déplacées hors de l’écran en mode plein écran.
* Infobulles refactorisées pour résoudre divers bogues.

**Bouton**

* Ajout de la prise en charge de CSS pour les styles de bouton par défaut.

**Conteneur**

* Correction : taille de conteneur incorrecte après le retour d’un affichage plein écran et la permutation des ressources.
* Correction : les coordonnées du conteneur sont incorrectes si le conteneur est déplacé avec une transformation CSS.

**EmailShare**

* Correction : artefact d’info-bulle se produisant avec le bouton Supprimer .
* Correction : les info-bulles de la barre de défilement s’affichent derrière l’interface utilisateur et sont recadrées.
* Correction : l’info-bulle « Annuler » apparaissait derrière le bouton Envoyer un e-mail .

**IncorporerPartager**

* Correction : le texte du code incorporé n’est pas renvoyé à la ligne et s’affiche sur la barre de défilement dans Internet Explorer.

**FlyoutZoomView**

* Correction : les événements Flyout ne sont pas envoyés sur Internet Explorer 8.

**Volume mutable**

* Fixe : l’info-bulle Volume s’affiche sous la barre de volume.

**Nuanciers**

* Correction : `selectSwatch` méthode ne se déclenche pas lorsqu’elle est incorporée dans la fonction `initViewer()`.

**SocialShare**

* Ajout de la prise en charge du modificateur de roulement pour contrôler l&#39;emplacement de disposition des boutons de partage.
* Ajout d’événements pour contrôler la pause/lecture lors de l’interaction avec les fonctionnalités sociales.
* Correction : l’icône Partage sur les réseaux sociaux ne disparaît pas avec le `controlbar` sur Zoom.
* Correction : l’icône Partage social n’apparaissait pas lors du fondu initial dans Galaxy SIII.
* Correction : les boîtes de dialogue modales de partage sur les réseaux sociaux ne s’affichent pas en mode plein écran sur iPad3.
* Correction : les boîtes de dialogue modales de partage sur les réseaux sociaux ne s’affichent pas en mode plein écran sur Mac avec Safari 5.1.
* Correction : les boîtes de dialogue modales Partage sur les réseaux sociaux s’affichaient hors de l’écran après un défilement.

**ThumbnailGridView**

* Correction : ThumbnailGridView n’envoie pas de `AssetEvent.ASSET_CHANGED`.

**VideoPlayer**

* Ajout de la prise en charge du modificateur `initialBitRate` pour sélectionner le débit utilisé pour la lecture initiale sur le bureau.
* Implémentation d’un proxy vidéo iPhone pour résoudre les problèmes vidéo sur les appareils iPhone.
* Ajout de la prise en charge de l’écouteur d’événement pour suspendre la lecture lorsque les fonctionnalités sociales sont appelées.
* Le mode de lecture défini sur auto utilise désormais le lecteur natif sur le bureau Safari.
* Correction : le protocole HTTPS générait un avertissement de sécurité sur IE8 à l’aide de la lecture Flash.
* Correction : la vidéo ne se dimensionne pas correctement en mode plein écran sur Safari 5 avec la lecture Flash.
* Correction : les nuanciers dans la visionneuse de médias ne peuvent pas être sélectionnés après la lecture de la vidéo sur iPhone.

**ZoomView**

* Correction : artefacts affichés après un zoom sur une image ou un appareil rotatif sur Android™.
