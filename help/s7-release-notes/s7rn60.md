# Notes de mise à jour de Scene7 Publishing System 6.0{#scene-publishing-system-release-notes}

Notes de mise à jour qui décrivent les fonctionnalités, les améliorations et les correctifs de Adobe Scene7 partie Publishing System 6.0 du Adobe Marketing Cloud.

## Nouvelles fonctionnalités et améliorations {#section-4bbd330df5a043f7a56eee619ef7c442}

**Généralités**

* Prise en charge du chinois.
* Améliorations en termes d’administration, permettant notamment d’effectuer les opérations suivantes :

   * Exporter la liste des utilisateurs de l’entreprise.
   * Envoyer des e-mails en copie carbone invisible aux utilisateurs.
   * Réinitialiser les mots de passe utilisateur sur un mot de passe aléatoire que les utilisateurs peuvent ensuite redéfinir.

* Schémas de métadonnées

  Les schémas de métadonnées définissent des ensembles de propriétés de métadonnées. Chaque propriété du schéma de métadonnées possède un type de données de base tel que String, Float, Integer, Date ou Yes/No. Il est aussi possible de définir des listes et des choix ouverts ou fermés. Les schémas de métadonnées peuvent être utilisés dans tous les workflows de métadonnées tels que la recherche, la modification, l’importation, les vues de métadonnées et les préréglages de métadonnées. <!-- [More information](http://help.adobe.com/en_US/scene7/using/WS259993e42159a215-1c6a66df1265272619e-7fec.html#WSd968ca97bf03cf72-5e3dd3a113268dc80f5-8000). -->

  Les schémas de métadonnées sont la technologie qui succède aux champs définis par l’utilisateur et sont destinés aux flux de travail de métadonnées personnalisés, en particulier dans le contexte du serveur de métadonnées.

* Serveur de métadonnées

  >[!NOTE]
  >
  >Le serveur de métadonnées est disponible à partir du mois de mai. Il doit être activé par le support. Le serveur de métadonnées n’est actuellement disponible qu’en Amérique du Nord et en Europe.

* Le serveur de métadonnées expose un sous-ensemble de tous les champs de métadonnées d’une ressource à un serveur public et peut être demandé ici par les pages ou serveurs eCom. Ces métadonnées exposées peuvent constituer un ensemble mixte de propriétés d’actif, de champs définis par les utilisateurs, de XMP et de propriétés de schéma de métadonnées. Le serveur de métadonnées offre une API Web avec une variété de fonctionnalités de recherche telles que des recherches de métadonnées, ou la similiarité de l’image, ou les deux. Pour chaque fichier trouvé, une liste de propriétés de métadonnées est demandée.
* La fonction d’importation de métadonnées peut gérer XMP propriétés

  L’importation de métadonnées est étendue pour gérer XMP propriétés et les propriétés définies dans Schémas de métadonnées.
* La vue des métadonnées affiche uniquement les champs contenant des valeurs.

  Il existe une nouvelle vue de métadonnées prédéfinie appelée « Toutes les propriétés avec valeurs » qui affiche uniquement les propriétés de métadonnées contenant des valeurs. Les propriétés (de XMP, des champs définis par l’utilisateur ou des nouveaux schémas de métadonnées) sans valeurs sont masquées et n’apparaissent pas.
* Les paramètres de métadonnées sont maintenant groupés dans l’interface utilisateur d’administration.

  Dans **[!UICONTROL Configuration > configuration de]** l’application, les pages de paramètres relatives aux métadonnées sont désormais regroupées dans une section de niveau supérieur appelée [!DNL Metadata]. Vous trouverez ici les pages de paramètres pour la vue des métadonnées, les paramètres prédéfinis des métadonnées, le schéma de métadonnées et les champs définis par l’utilisateur.

Cette application de bureau vous permet d’importer des modèles 3D existants et de les créer en tant que fichiers de vignettes 2D. Les fichiers de vignettes résultants peuvent être téléchargés vers SPS, où les couleurs et les textures peuvent être remplacées à la demande.

## Correctifs logiciels {#section-4219d5dd0a024cd3a95aaac45598363c}

Les problèmes suivants ont été corrigés sur le serveur de métadonnées :

* Le problème à l’origine de l’échec des envois de fichiers EPS très volumineux est désormais résolu.
* Les « recherches de métadonnées » avec de longues listes de résultats échouaient sans générer de message d’erreur. Ce problème était à l’origine de l’affichage de mauvais résultats de recherche.
