## Scene7 IPS (Image Production System) 4.5.0 {#section-3e37ed37b6d146419f4578bd7ffeaa74}

**Nouvelles fonctionnalités**

* En cours d’exécution via JDK 1.8.
* Prise en charge de la balise d’orientation EXIF.
* Téléchargez le fichier d’annuaire ignoré pour les fichiers FTP temporaires (comme créé avec ProFTP).
* La publication instantanée est désormais prise en charge pour le rendu d’image. Cela signifie que vos fichiers de vignette rendus sont immédiatement disponibles pour le lancement sur votre site Web, sans avoir à lancer une tâche de publication. Si vous souhaitez utiliser cette fonctionnalité, contactez le support technique pour activer votre compte pour une publication instantanée.

**Fonctions non prises en charge**

* Option de resynchronisation du serveur de catalogue dans la validation de publication classique IPS - supprimée.
* Vous ne pouvez plus définir de valeurs de champ de balise de métadonnées vides ou  vides.

**Correctifs**

* Le champ `<metadataArray>` était vide pour le [!DNL .vtt] fichier en réponse à l’API `getAssets`.

* Champ userdata manquant pour l’orientation.
* La réponse `getJobLogDetails` d’une tâche mettant à jour un fichier ACO ne spécifiait pas correctement les détails de la tâche.
* Le fichier principal PSD a été enregistré plusieurs fois comme publié par Image Server.
* Les fichiers dont le nom contient des caractères non valides peuvent être téléchargés dans IPS.
* Les miniatures vidéo qui n’ont pas pu être générées ont été laissées dans l’état &quot;En attente dans la file d’attente&quot;.
* Condition de concurrence avec la restauration des dossiers entre le téléchargement des répertoires volumineux `deleteFolder` et serveur.
* Les fichiers PDF étaient consignés deux fois par publication par serveur de publication lors de leur publication initiale au moment du téléchargement.
* Le téléchargement de plusieurs fichiers PSD portant les mêmes noms de calque générait des erreurs d’optimisation.
* La pixellisation PDFL du fichier PDF client générait une ligne blanche inattendue dans l’image générée.
* Les fichiers de miniatures vidéo ont été publiés plusieurs fois au moment du téléchargement.
* La détection des fichiers de téléchargement  des a généré deux notifications de téléchargement, supprimant la comptabilisation des fichiers terminée.
* Le fichier XML sidecar pour le transcodage vidéo a été enregistré comme publié plusieurs fois lors de la mise à jour.
* moveFolder a échoué lorsque vous avez supprimé le dossier imbriqué et déplacé un autre dossier (qui ne se trouve pas dans le dossier parent du dossier imbriqué et porte le même nom que le dossier imbriqué) vers l’emplacement où se trouvait le dossier imbriqué.
* Les entrées de configuration d’état de cluster dans les configurations partagée et locale n’ont pas été purgées par nettoyage.
* Modifiez l’analyseur de journaux de l’API pour filtrer les formats d’adresse électronique.
* Le traitement vidéo n’est pas mappé `Iptc4xmpCore:Location` correctement.
* La condition de race existait entre le nettoyage et le transfert.
* La publication n’est pas utilisée `assetFile` de sorte que la publication intermédiaire fonctionne uniquement pour les nouveaux actifs.
* Pointeur nul généré lors de la mise à jour d’un jeu de règles.

## Visionneuses {#section-833ab92c91c941d2bfdc27f233f582ad}

**Nouvelles fonctionnalités, améliorations et corrections de bogues pour les visionneuses Scene7 Image Serving 5.2.2**

Les mises à niveau du lecteur sont rétrocompatibles ; aucune modification n’est nécessaire pour votre code Web existant. Toutefois, il est recommandé de tester les nouvelles visionneuses sur notre  d’évaluation. Le site Web suivant explique comment configurer votre système pour accéder à notre serveur d’évaluation :

[http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html](http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html)

Après avoir configuré votre ordinateur pour accéder au serveur d’évaluation, vous pouvez consulter votre site Web pour tester la mise à niveau. Si vous utilisez des visionneuses HTML5 prêtes à l’emploi, il est recommandé de les tester par rapport à notre serveur d’évaluation standard et à s7is5--staging.scene7.com.

Nouvelles fonctionnalités et améliorations

Voir aussi le Guide [de référence des visionneuses](https://marketing.adobe.com/resources/help/en_US/s7/viewers_ref/)Scene7.

* Ajout de la prise en charge de l’impression, du téléchargement et des favoris dans la visionneuse de catalogue électronique.
* Possibilité de récupérer ParameterManager à l’aide de l’API getComponent.
* Visionneuse à 360° convertie, visionneuse de zoom, visionneuse vidéo et visionneuse déroulante pour utiliser des images-objets pour les illustrations.
* Ajout de la prise en charge du mode plein écran natif d’IE11.
* Prise en charge en plein écran simulée (non native) refactorisée dans les  de.
* Augmentation de la taille des petits marqueurs CSS pour prendre en charge les téléphones plus volumineux.
* Suppression des marqueurs de taille CSS pour les visionneuses à 360° HTML5, BasicZoom, Zoom, Spin et MixedMedia sur les navigateurs de bureau.
* Ajout de la prise en charge de la configuration de qualité des images préchargées dans SpinView.
* Fixe - Galaxy S4, mode portrait : Marqueur de taille CSS incorrect utilisé lorsque le mode plein écran était activé.
* Corrigé : les zones cliquables [du catalogue] électronique sont bloquées après l’utilisation de modèles JavaScript pour la liaison externe sur Internet Explorer 9.
* Corrigé : Internet Explorer 9 et Internet Explorer 10 : le zoom ne fonctionne pas correctement si la propriété width est définie pour les IMG dans CSS.
